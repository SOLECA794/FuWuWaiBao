# 项目简介：基于泛雅平台的 AI 互动智课生成与实时问答系统

## 1. 项目定位
本项目面向高校通识课“自主学习”场景，提供一套可嵌入泛雅 Web/学习通 WebView 的 AI 互动智课能力，实现从“静态课件（PPT/PDF）”到“可讲授、可打断、可续接”的学习闭环。

一句话描述：
- **把 PPT/PDF 变成能讲课的 AI 学伴**；学生可随时打断提问，系统结合课件上下文精准回答，并按理解程度自动续接/补充讲解。

## 2. 场景适配性分析（为什么适配泛雅/高校通识课）

### 2.1 典型教学场景
- 教师端：教师上传课件（PPT/PDF），希望减少“录讲解视频”的时间成本，同时能对讲稿进行可控编辑。
- 学生端：学生在自主学习中容易出现“理解断点”，需要即时答疑与可续接讲授。
- 平台端：泛雅需要标准化、可扩展的开放接口，支持异步任务对接与多端嵌入。

### 2.2 设备与网络适配
- 目标设备：常规 PC / 移动端（骁龙 855 及以上、4G 内存）。
- 关键策略：
  - 解析任务异步化（避免主链路阻塞，≤2 分钟/份）
  - 问答采用 SSE 流式输出（首 token 快返回，体感 ≤5 秒）
  - 大模型能力外部化（服务器 8C16G 约束下不本地常驻大模型）

### 2.3 教育场景的“答案可信度”需求
- 采用“本页优先 + 溯源页码/章节”的 grounded QA 输出，前端可展示答案来源，降低幻觉风险。

## 3. 系统能力概览（面向评审/对接方）

### 3.1 智课生成
- PPT/PDF 多模态解析：识别知识点层级、公式/图表要点、重点标注。
- 结构化讲授脚本：按页/节点输出 `opening/explain/transition` 脚本，支持教师编辑。
- 可扩展：语音合成（TTS）/数字人讲授接口预留。

### 3.2 实时问答（文字/语音）
- 上下文关联问答：携带当前页/章节/脚本节点进行检索增强生成（RAG）。
- 多轮交互：支持 sessionId 维持上下文。
- 流式体验：SSE `token/sentence/final/error`。

### 3.3 进度续接与节奏调整
- 断点续接：精确到 `page + nodeId`。
- 理解度评估：输出 `understandingLevel`（none/partial/full）与 `need_reteach`。
- 节奏策略：根据理解度返回 `adjustType`（supplement/accelerate/normal）。

## 4. 与泛雅平台集成方案（关键）

### 4.1 集成形态
- **前端嵌入**：学生端/教师端以 Web 页面形式嵌入泛雅（Web/学习通 WebView）。
- **接口对接**：平台侧通过开放 API 调用课件解析、脚本生成、问答交互与进度追踪。

### 4.2 双接口层策略（兼容性与演进性）
- 内部产品 API：给我们的前端使用（字段更贴合前端，见 `API_DESIGN_V2.md` 第 2~7 章）。
- 泛雅 OpenAPI 兼容层（OpenAPI Adapter）：严格按泛雅规范（见 `泛雅API规范.md` 与 `API_DESIGN_V2.md` 第 8 章）。
  - URL 版本：`/api/v1/...`
  - 签名：`enc=MD5(参数有序拼接+staticKey+time)`
  - 统一响应：`code/msg/data/requestId`

### 4.3 异步任务对接
- `lesson/parse`、`lesson/generateScript` 为耗时任务：返回 `parseId/scriptId` 与 `taskStatus`。
- 平台侧可轮询任务状态，或按后续协商扩展回调通知。

### 4.4 安全与合规
- 签名验证 + HTTPS 传输。
- 最小必要数据原则：仅存储必要的学习进度、问答记录；敏感信息可脱敏/加密。
