## 1. 项目交付切分（建议按 3 个里程碑推进）

### M1（可跑通闭环的 MVP）
**目标**：教师上传课件 → 解析生成页级脚本 → 学生能看页预览+脚本 → 文字问答（非语音）能返回答案并能“续接”。

**必须具备**：
- 课件上传与存储（MinIO）
- 解析任务异步化（Redis 队列）
- 页级脚本 JSON 能被教师端编辑保存
- 学生端问答走 SSE，能看到流式文本（打字机）
- 问答后能恢复到原 page/node

### M2（性能与体验达标）
**目标**：问答体感接近 ≤5s；引入分句事件（sentence）支持 TTS；解析稳定性提升。

**必须具备**：
- SSE 事件帧：token/sentence/final/error
- Go 侧问答代理的超时、断线重连策略
- 解析限并发（8C16G 保护）

### M3（语音与进度节奏调整）
**目标**：语音打断（ASR）+ 语音回答（TTS）+ 重讲判定 need_reteach 带节奏调整。

## 2. 角色分工（每个角色的“交付物”）

### 2.1 后端（Go）交付物清单
1. **异步解析任务调度**
   - 入队：上传后写库 + 推送 ParseJob 到 Redis
   - 出队：Python worker 消费（或 Python HTTP 回调）
   - 状态：parsing/ready/failed 可查询

2. **学生端问答 SSE 入口**
   - `POST /student/qa/stream`：
     - 入参：question + current_page + current_node_id
     - 出参：SSE（token/sentence/final/error）
   - 关键：边到边转发（不能等 Python 输出完再返回）

3. **学习进度状态机（Redis）**
   - 维护字段：current_page/current_node_id/interrupted/resume_* 等
   - DoD：断点续接可复现（手动验收）

4. **数据落库与日志**
   - QuestionLog：记录每次问答的 question/answer/source_page

### 2.2 AI（Python）交付物清单
1. **解析 + 生成（离线可验收）**
   - `ai_engine/parser.py`：输出统一 schema
   - `ai_engine/generate.py`：输出每页 nodes[]（opening/explain/transition）
   - DoD：`ai_engine/debug_harness.py` 全绿（或 summary.json 达标）

2. **问答（RAG + 溯源 + 重讲判定）**
   - 输入：question + page_index +（可选）history
   - 输出：answer/source_page/resume_page/resume_node_id/need_reteach
   - 流式输出：能持续 yield token/sentence

3. **数据契约稳定**
   - 对脚本 nodes[] 字段不得随意改名
   - 任何字段变更必须同步更新文档

### 2.3 前端（教师端）交付物清单
1. 课件管理：上传/列表/删除/发布
2. 按页讲稿编辑：加载/编辑/保存（对应 API.md）
3. 解析状态展示：解析中/成功/失败 + 重试入口（若后端支持）

### 2.4 前端（学生端）交付物清单
1. 播放器：显示页预览 + nodes 高亮
2. 问答：输入框 + SSE 展示（打字机）
3. 续接：问答前记录 current_page/node_id，final 事件后恢复播放
4. （M3）语音：ASR 输入、TTS 分句播放

## 4. 联调方法（建议固定节奏）

### 4.1 第一次联调前必须“冻结契约”
冻结 3 件事：
1. 页级脚本 JSON（nodes[] 字段）
2. SSE 事件帧（token/sentence/final/error）
3. 问答 final 输出字段（need_reteach/resume_page/resume_node_id/source_page）

冻结后，任何变更走 PR + 文档同步。

### 4.2 联调顺序（从最短链路开始）
1. Go SSE → 前端（先不接 Python，直接写死 token）
2. Python QA 流式 → Go（命令行或本地 HTTP）
3. Go 透传 → 前端
4. 再把 Session 状态机接上

---

## 5. 质量与验收（DoD 统一口径）

- 功能 DoD：按照“任务拆解表”的 DoD 逐条过
- 性能 DoD：
  - 问答：首 token ≤ 2s（体感），完整回答 ≤ 5s（网络良好时）
  - 解析：单份 ≤ 2min（限并发后测）
- 可观测性 DoD：
  - 每次解析任务有 task_id
  - 每次问答有 trace_id（可用 request_id 替代）
