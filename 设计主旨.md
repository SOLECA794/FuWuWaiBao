作为资深 UI/UX 设计师，为「教师端课件编辑后台」设计高保真界面，**完整保留所有功能结构与按钮文案**，严格遵循「Apple Human Interface Guidelines 为主、Material Design 3 为辅」的核心设计规范，在保证专业克制、高效易用的基础上，增加有叙事感、有张力但绝不浮夸的动画交互效果，所有动效 100% 服务于功能逻辑，不做无意义的装饰性动画。

## 一、基础 UI 设计铁则（先保证界面不丑、符合规范）

1. **层级规范**：严格遵循「页面背景 #F5F7FA → 侧边栏 #FFFFFF → 主窗口卡片 #FFFFFF → 内容块 #F8FAFC」的四层灰度层级，双窗口卡片用 16px 统一大圆角 + 极淡弥散阴影`0 4px 20px rgba(0,0,0,0.06)`，悬浮感明确，和页面背景拉开清晰层级；
2. **统一规范**：所有元素严格遵循 8px 栅格系统，圆角体系固定（大容器 16px、内容块 12px、按钮 / 标签 8px、主按钮 12px 胶囊全圆角），控件样式、字号字重、图标粗细完全统一，所有元素严格对齐；
3. **色彩规范**：低饱和淡紫色`#7C3AED`为唯一主题色，界面占比≤10%，仅用于主按钮、选中态、重点强调；灰度层级严格区分（标题 #1E293B、正文 #334155、辅助文字 #64748B），无高饱和花哨配色；
4. **排版留白**：字体层级固定（窗口标题 18px 600 字重、副标题 14px 500 字重、正文 14px 400 字重、辅助文字 12px 400 字重），模块间距≥24px，内容内边距统一，给足呼吸感，无拥挤混乱。

## 二、动画交互核心准则（保证不破坏设计原则的底线）

1. **功能优先**：所有动效必须服务于「用户理解功能、感知操作反馈、明确层级关系」，无任何纯装饰性、无意义的动画；
2. **克制高效**：所有基础动效时长严格控制在**0.2s-0.35s**，仅叙事性入场动效最长不超过 0.5s，缓动函数统一使用苹果原生的`easeOut`（先快后慢，符合物理惯性），绝不拖沓、不影响操作效率；
3. **层级叙事**：动效严格跟随界面层级，有明确的先后顺序和节奏差，通过动效强化「谁在上、谁在下、谁是核心、谁是辅助」的空间关系，打造视觉张力；
4. **反馈精准**：用户的每一次点击、hover、切换操作，都必须有对应的、即时的动效反馈，让用户明确感知「操作已生效」，同时强化交互的愉悦感。

## 三、分模块精准动效要求（有张力、不浮夸，可直接落地）

### 1. 页面入场动效（打造第一视觉张力，强化层级）

* 触发时机：页面首次加载 / 刷新
* 动效逻辑：按「从外到内、从底层到顶层」的顺序错落入场，有明确的节奏差，而非所有元素同时出现
* 具体效果：
  1. 页面背景与侧边栏：0.2s 淡入，无位移，先完成渲染；
  2. 双窗口主卡片：延迟 0.1s 触发，从下往上 20px 位移 + 淡入，0.3s 完成，两个窗口错落 0.05s 先后入场，强化双容器的独立感；
  3. 窗口内的标题、内容块、按钮：延迟 0.2s 触发，从下往上 10px 位移 + 淡入，0.25s 完成，内容块错落 0.03s 先后入场，强化内部层级。

### 2. 容器交互动效（强化悬浮感，打造微张力）

* 触发时机：鼠标 hover 窗口卡片 / 内容块
* 动效效果：0.2s 平滑过渡，卡片从`translateY(0)`变为`translateY(-4px)`，阴影从`0 4px 20px rgba(0,0,0,0.06)`变为`0 8px 30px rgba(124, 58, 237, 0.08)`，轻微放大 1.01 倍，松手后顺滑回弹，强化卡片的可交互质感。

### 3. 基础控件反馈动效（即时响应，强化愉悦感）

所有可点击元素（按钮、标签、图标、关闭按钮）必须有统一的三级反馈动效：

1. **Hover 态**：0.15s 平滑过渡，按钮 / 标签轻微放大 1.03 倍，背景色加深 10%，图标同步变色，无生硬跳转；
2. **点击态**：0.1s 快速过渡，元素轻微缩小 0.97 倍，模拟物理按压反馈，松手后立即回弹；
3. **选中态**：0.25s 平滑过渡，选中标签的背景色从透明变为主题色填充，文字从深灰变为白色，带同色系的轻微底色扩散动效，明确选中状态。

### 4. 功能转场动效（服务核心操作，有叙事张力）

针对课件编辑的核心场景，做有意义的转场动效，强化操作逻辑：

1. **标签切换动效**：点击顶部「课件预览窗口 / 内容编辑器窗口」标签时，0.25s 完成，未选中标签淡出 + 轻微缩小，选中标签淡入 + 放大，下方对应窗口同步做淡入淡出 + 10px 位移的顺滑切换，无生硬闪切；
2. **课件节点切换动效**：点击底部节点 / 翻页按钮时，预览窗口内容做「淡入淡出 + 左右 15px 滑入滑出」的转场动效，0.3s 完成，进度条同步做顺滑的填充动效，明确页面切换的逻辑；
3. **侧边栏导航切换**：点击侧边栏菜单时，0.2s 完成，选中项的左侧主题色指示条做顺滑的位置滑动动效，同时选中项背景色淡入，明确当前所在页面。

### 5. 状态反馈动效（强化操作结果，微张力不打扰）

针对操作结果做轻量化的反馈动效，不打断用户操作：

1. **保存成功动效**：点击「保存讲稿与节点」按钮后，按钮先做按压反馈，再变为「对勾 + 保存成功」的文字，带轻微的主题色底色扩散动效，0.3s 完成，2s 后顺滑回弹，无需弹窗打断；
2. **进度变动效**：课件播放 / 节点切换时，底部进度条的滑块做顺滑的线性填充动效，节点切换时滑块带轻微的弹性回弹，明确进度变化；
3. **新增 / 删除节点动效**：点击「新增节点」时，新节点卡片从下往上淡入 + 展开，0.3s 完成；删除节点时，卡片向上收缩 + 淡出，周围元素顺滑补位，无生硬跳转。

## 四、绝对禁止项（防止动效破坏设计原则与产品体验）

1. 禁止使用超过 0.5s 的长时长动效、循环动画、闪烁特效、3D 翻转 / 旋转等浮夸特效；
2. 禁止使用高饱和渐变、发光、粒子特效等破坏界面专业度的装饰性动效；
3. 禁止动效影响用户操作效率，所有动效不可打断用户的连续输入、点击操作；
4. 禁止动效破坏界面层级规范，不可出现元素层级混乱、遮挡核心内容的情况。
